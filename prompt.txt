## Feature Engineering (Notebook 2)

Load `raw_races.parquet`, `raw_horses.parquet`, `weather.parquet`. Create a final feature matrix for supervised learning (binary classification: horse wins race or not).

**Steps:**
1. Clean and merge:
   - Convert date columns to datetime.
   - Merge horse static data onto each runner row (by brand number). For age, compute age at race date (using profile's birth year or subtract from race date).
   - Merge weather data by date and racecourse.

2. Create time‑ordered dataset per horse: sort by horse and race date. Use `groupby('horse_brand_no')` and `shift()` / `rolling()` to compute **past performance features with no look‑ahead**:
   - Last 3 finishes: average finishing position, win rate, place rate.
   - Last 5 runs: average distance beaten, average odds, average rating change.
   - Last 2 runs: average speed (distance / finish time).

3. Jockey & trainer rolling statistics (from all previous races, not just that horse):
   - For each race, compute for the jockey: win rate at same (course, distance) in last 30 days; overall win rate last 30 days.
   - Same for trainer.
   - These must be computed incrementally. Approach: sort all races by date, maintain a daily‑updated lookup table, then map onto each race row.

4. Derived features:
   - `is_sprint` = distance < 1400m, `is_mile` = 1400‑1600m, `is_staying` > 2000m.
   - `draw_normalised` = draw / (max_draw_for_race) – but max draw varies by course/distance; you can use fixed typical max (14).
   - `weight_for_age` = actual_weight / (horse_age * 2 + 50) – rough normalisation.
   - `speed_mps` = distance_m / finish_time_sec (if finish_time available).
   - `early_speed` = sectional_400m time (lower is faster).
   - `finish_speed` = last 400m sectional (or last 800m minus first 400m).
   - `implied_prob` = 1 / win_odds (clip odds to avoid division by zero).
   - Weather: `rain_24h_before` (rainfall on race day), `temp_c`, `humidity`.

5. Encode categoricals:
   - One‑hot encode `going` (Good, Good to Firm, Yielding, etc.) and `course_type`.
   - Target encode `jockey` and `trainer` with smoothing (using mean win rate from training set only – careful with time split). Alternatively, use leave‑one‑out encoding within each time window.

6. Target variable: `is_winner` = 1 if finish_position == 1 else 0.

7. Split data **by date** (no random split):
   - Train: dates < (latest_date - 6 months)
   - Validation: (latest_date - 6 months) to (latest_date - 3 months)
   - Test: last 3 months of data
   Use actual date ranges based on your data. Save splits as `X_train.parquet`, `y_train.parquet`, etc.

**Pitfall checks:** Ensure that rolling features for a race only use races that occurred strictly before that race's date. Use `shift(1)` after sorting.

## Model Training & Evaluation (Notebook 3)

**Objective:** Predict win probability for each horse in a race. Evaluate on both ranking metrics and simulated betting profit.

**Models to implement:**
- Baseline: Logistic Regression (with class_weight='balanced')
- Random Forest (with class_weight='balanced')
- XGBoost (scale_pos_weight = (negative_count/positive_count))
- LightGBM (is_unbalance=True or scale_pos_weight)

**Training setup:**
- Use the training split only for fitting. Tune hyperparameters on validation split.
- For XGBoost/LightGBM, use `Optuna` or `GridSearchCV` with 3‑fold time‑series cross‑validation (e.g., TimeSeriesSplit from sklearn).
- Evaluation metrics: log‑loss, AUC, and **ranking metrics** – for each race, rank horses by predicted probability, compute Mean Average Precision @3 (MAP@3) and Normalized Discounted Cumulative Gain @3 (NDCG@3).

**Probability calibration:**
- After training, apply Platt scaling (LogisticRegression on validation set's predicted probabilities vs true labels) or Isotonic Regression to calibrate probabilities. Use `sklearn.calibration.CalibratedClassifierCV` with cv='prefit'.

**Betting backtest (profitability simulation):**
Simulate betting on the test set (races in the test date window). For each race:
- For each horse, compute calibrated win probability `p` and final win odds `odds` (from scraped data).
- Expected value (EV) = `p * (odds - 1) - (1 - p)`.
- Bet only if:
   - EV > 0.05 (5% edge)
   - `p > 0.15` (avoid extremely long shots)
   - Horse is among top 2 predicted probabilities in the race (optional filter)
- Bet size: fixed fraction of current bankroll (e.g., 1% per bet). Starting bankroll = 1000 units.
- Track each bet: stake, outcome (win/loss), net profit. Compute daily or per‑race returns.

**Output metrics for each model:**
- Accuracy, AUC, log‑loss on test set.
- MAP@3, NDCG@3.
- Total bets placed, win rate on bets, ROI = (net profit / total staked) * 100%, final bankroll, Sharpe ratio (if daily returns available).

**Comparison and selection:**
- Print a summary table comparing all models.
- Select the model with highest ROI (if positive) and reasonable bet win rate (>25% on bets). If no model is profitable, suggest adjusting EV threshold.

**Optional extensions (if time):**
- Ensemble: average probabilities from top 3 models (XGBoost, LightGBM, Random Forest) and repeat backtest.
- Simple neural net (PyTorch, 2‑layer with dropout) – include as an extra model.
